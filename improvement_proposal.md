# 既存システムとの比較に基づく改善提案書

**作成日**: 2025年12月11日  
**作成者**: Manus AI

---

## 🎯 目的

この提案書は、既存システム（カイポケ、かべなしクラウド）との比較分析に基づき、現在のシステムが劣っている部分を特定し、その機能を補完するための具体的な改善策を提案します。これにより、「既存システムに劣る部分がゼロ」の状態を目指します。

---

## ⚡️ 改善提案

### **提案1: タブレット完全対応（現場記録の効率化）**

| 項目 | 内容 |
|---|---|
| **現状** | PCブラウザでの利用が前提。現場での記録入力に課題。 |
| **競合** | カイポケ、かべなしクラウド共にタブレット対応済み。 |
| **重要性** | 職員が支援の現場で直接記録を入力できるため、業務効率が飛躍的に向上し、記録のリアルタイム性が確保される。 |
| **実装案** | フロントエンドのUIをレスポンシブデザインに完全対応させ、タブレットやスマートフォンでの操作性を最適化する。特に記録入力画面のボタンサイズやレイアウトを調整する。 |

### **提案2: 電子サイン機能（ペーパーレス化の実現）**

| 項目 | 内容 |
|---|---|
| **現状** | 帳票はPDF出力後、印刷して署名・捺印が必要。 |
| **競合** | かべなしクラウドは電子サイン機能を搭載。 |
| **重要性** | 個別支援計画書や実績記録票への同意署名を電子化することで、ペーパーレス化を推進し、印刷・保管コストを削減。コンプライアンスも強化される。 |
| **実装案** | 1. **ライブラリの導入**: `react-signature-canvas` などの署名パッドライブラリをフロントエンドに導入。 <br> 2. **署名データの保存**: 署名データを画像として保存し、PDF生成時に埋め込む。 <br> 3. **APIの拡張**: 署名データを受け取り、帳票と紐づけて保存するAPIを開発。 |

### **提案3: メール交付機能（連絡業務の自動化）**

| 項目 | 内容 |
|---|---|
| **現状** | 生成した帳票は手動でメールに添付して送信する必要がある。 |
| **競合** | かべなしクラウドはメール交付機能を搭載。 |
| **重要性** | 個別支援計画書や請求書などの帳票をシステム内から直接保護者や関係機関にメールで送信できるようにし、連絡業務の手間を削減する。 |
| **実装案** | 1. **メール送信サービスの利用**: SendGridなどの外部メール送信サービスと連携。 <br> 2. **APIの開発**: 宛先、件名、本文、添付ファイル（生成したPDF）を指定してメールを送信するAPIを開発。 <br> 3. **UIの追加**: 利用者詳細画面や帳票出力画面に「メールで送信」ボタンを追加。 |

### **提案4: 工賃計算機能（就労支援事業所への対応）**

| 項目 | 内容 |
|---|---|
| **現状** | 就労支援A型・B型事業所で必須となる工賃計算機能がない。 |
| **競合** | かべなしクラウドは工賃計算機能を搭載。 |
| **重要性** | 対応できる事業所の範囲を就労支援領域まで拡大し、システムの市場価値を高める。 |
| **実装案** | 1. **モデルの追加**: 作業内容、作業時間、単価などを記録するモデルを追加。 <br> 2. **計算ロジックの実装**: 利用者ごと、作業ごとに工賃を自動計算するロジックを開発。 <br> 3. **APIとUIの追加**: 工賃の登録・集計・管理を行うためのAPIとUIを開発。 |

### **提案5: 送迎管理機能（通所系サービスへの対応）**

| 項目 | 内容 |
|---|---|
| **現状** | 通所系サービスで必要な送迎表の作成や管理機能がない。 |
| **競合** | かべなしクラウドは送迎表機能を搭載。 |
| **重要性** | 通所介護や放課後等デイサービスなど、多くの事業所で必須となる機能を搭載し、対応範囲を広げる。 |
| **実装案** | 1. **モデルの追加**: 送迎ルート、担当ドライバー、乗車する利用者などを管理するモデルを追加。 <br> 2. **UIの開発**: ドラッグ＆ドロップで簡単に送迎ルートを作成・編集できるUIを開発。 <br> 3. **PDF出力**: 最適化された送迎表をPDFで出力する機能を追加。 |

---

## 📊 提案の優先順位

以下の優先順位で実装を進めることを提案します。

| 優先度 | 提案 | 理由 |
|---|---|---|
| **1** | **タブレット完全対応** | 全ての事業所、全ての機能に影響する最も基本的な改善。 |
| **2** | **電子サイン機能** | ペーパーレス化の需要が非常に高く、導入効果が大きい。 |
| **3** | **メール交付機能** | 連絡業務の効率化は、日々の運用負担軽減に直結する。 |
| **4** | **工賃計算機能** | 就労支援という大きな市場セグメントを獲得するために必須。 |
| **5** | **送迎管理機能** | 通所系サービスという主要な市場セグメントに対応するために必要。 |
